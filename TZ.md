# Техническое задание на проект: Debt Solver

## 1. Цель проекта
Создание веб-сервиса для оптимизации погашения задолженности по банковским картам. Сервис помогает физическим лицам с 2–5 кредитными картами рассчитать реальную переплату и построить оптимальный график платежей, минимизируя процентные расходы.

## 2. Роли пользователей
- **Гость (неавторизованный пользователь)**: вводит данные по своим кредитным картам (до 5 штук), получает расчёт оптимального графика платежей и визуализацию переплаты.
- **Администратор**: через стандартную Django Admin панель управляет данными, просматривает историю расчётов, имеет доступ к агрегированной статистике.

*Авторизация обычных пользователей не предусмотрена — сервис одноразового использования без хранения персональных данных.*

## 3. Модели данных

### Модель 1: CreditCard (Кредитная карта)
Хранит параметры одной кредитной карты, введённые пользователем.
- `name` (CharField) — название карты или банка
- `credit_limit` (DecimalField) — кредитный лимит
- `interest_rate` (DecimalField) — годовая процентная ставка (%)
- `current_debt` (DecimalField) — текущая задолженность
- `min_payment_percent` (DecimalField) — минимальный платёж в % от долга

### Модель 2: PaymentSchedule (График платежей)
Хранит помесячный график погашения для каждой карты.
- `card` (ForeignKey к CreditCard) — связь с картой
- `month` (IntegerField) — номер месяца от начала погашения
- `payment_amount` (DecimalField) — сумма платежа в этом месяце
- `interest_paid` (DecimalField) — сумма процентов, начисленных за месяц
- `remaining_debt` (DecimalField) — остаток долга после платежа

### Модель 3: CalculationHistory (История расчётов)
Сохраняет результаты каждого расчёта для аналитики и возможности вернуться к результату.
- `session_id` (CharField) — идентификатор сессии (для гостей)
- `created_at` (DateTimeField) — дата и время расчёта
- `cards_snapshot` (JSONField) — снимок данных всех карт на момент расчёта
- `total_overpayment` (DecimalField) — общая переплата за весь период
- `payoff_months` (IntegerField) — количество месяцев до полного погашения
- `recommended_strategy` (TextField) — текстовое описание стратегии

## 4. Ключевой функционал

1. **Ввод данных карт**: Пользователь заполняет форму с параметрами от 2 до 5 кредитных карт (название, лимит, ставка, долг, % минимального платежа).

2. **Валидация данных**: Система проверяет, что суммарный долг не превышает суммарный кредитный лимит, и выводит понятные сообщения об ошибках.

3. **Расчёт оптимального графика**: Система применяет жадный алгоритм (сортировка карт по убыванию ставки, направление свободных средств на карту с максимальной ставкой) и генерирует помесячный график платежей.

4. **Визуализация результата**: Пользователь видит интерактивный график (Plotly) снижения общего долга по месяцам и таблицу с детальным графиком платежей по каждой карте.

5. **Сохранение истории**: Каждый расчёт автоматически сохраняется в историю. Администратор может просматривать агрегированную статистику в админ-панели (средняя переплата, средний срок погашения).

## 5. Внешние интеграции и аналитика

**Интеграция / Аналитика**: Библиотека **Plotly** для построения интерактивных графиков снижения долга по месяцам. Дополнительно используется **Pandas** для агрегации данных (расчёт средней переплаты, минимального и максимального срока погашения в админ-панели).

*API интеграции не требуется — все данные вводятся пользователем вручную через форму.*
